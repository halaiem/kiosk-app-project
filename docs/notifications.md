# Уведомления и сообщения на планшете водителя

---

## Типы уведомлений и сообщений

### 1. Обычное сообщение (`normal`)
- **От кого:** диспетчер, система
- **Вид:** всплывающий тост снизу, нейтральный цвет
- **Кнопки:** «Принято», «Ответить», «Да», «Нет»
- **Звук:** мягкий бип
- **Поведение:** не исчезает сам, ждёт действия водителя

### 2. Сообщение диспетчера (`dispatcher`)
- **От кого:** диспетчер
- **Вид:** тост с синеватым фоном, иконка радио
- **Кнопки:** «Принято», «Ответить»
- **Звук:** мягкий бип

### 3. Важное сообщение (`important`)
- **От кого:** диспетчер / администратор
- **Вид:** полноэкранный оверлей с пульсирующей красной рамкой
- **Заголовок:** «⚠️ ВАЖНОЕ СООБЩЕНИЕ» на красном фоне
- **Особенность:** фиксирует время реакции водителя (счётчик секунд)
- **Кнопки:** «Принято», «Да», «Нет», «Ответить»
- **Звук:** тревожный бип

### 4. Алерт диспетчера (`alert`)
- **От кого:** диспетчер
- **Вид:** карточка с анимированной пульсирующей иконкой, цвет меняется по типу:

| Цвет | Пример ситуации |
|------|-----------------|
| Красный | Вернуться в парк — техпроверка |
| Оранжевый | Объезд — ДТП на маршруте |
| Синий | Задержаться на конечной |
| Фиолетовый | Ревизор на маршруте |
| Жёлтый | Обрыв контактной сети |
| Зелёный | Голосовое сообщение |

### 5. Голосовое сообщение (`isVoice`)
- **От кого:** диспетчер
- **Вид:** зелёный тост с кнопкой «Play» и длительностью в секундах
- **В мессенджере:** плеер с прогресс-баром, кнопка транскрибации (расшифровка текстом)
- **Авто-воспроизведение:** да, при открытом мессенджере

### 6. Ошибка CAN-шины (`can_error`)
- **От кого:** система / техник (автоматически от борта)
- **Вид:** тост с оранжевым/жёлтым фоном, иконка предупреждения

### 7. SOS-сигнал
- **От кого:** водитель → диспетчеру (исходящий)
- **Вид:** красное модальное окно с пульсацией
- **Текст:** «SOS! Водитель вызывает диспетчера — СРОЧНО!»
- **После отправки:** зелёная галочка, автозакрытие через 3 сек

---

## Как доставляются

| Канал | Интервал | Описание |
|-------|----------|----------|
| Polling API | каждые 45 сек | Новые сообщения с сервера |
| Heartbeat | каждые 60 сек | Позиция и скорость отправляются на сервер |
| Offline-очередь | при восстановлении сети | Накопленные сообщения отправляются пакетом |

---

## Кто отправляет (роли)

| Роль | Тип сообщений |
|------|--------------|
| Диспетчер | normal, dispatcher, important, alert, voice |
| Система / API | can_error, автоматические уведомления |
| Администратор | important (через сервер) |
| Техник | can_error (бортовая система) |

---

## Очередь уведомлений

- Все уведомления попадают в единую очередь
- Показывается только верхнее (первое) уведомление
- Если уведомлений больше одного — отображается счётчик «N уведомлений»
- После подтверждения или ответа — следующее уведомление из очереди

---

## Поля сообщения (API)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Уникальный ID |
| `type` | normal / important / can_error / dispatcher | Тип сообщения |
| `text` | string | Текст сообщения |
| `timestamp` | Date | Время получения |
| `read` | boolean | Прочитано ли |
| `confirmed` | boolean | Подтверждено ли водителем |
| `severity` | info / warning / critical | Уровень важности |
| `isVoice` | boolean | Голосовое сообщение |
| `voiceDuration` | number | Длительность голосового (сек) |
| `transcription` | string | Расшифровка голосового |
| `audioUrl` | string | Ссылка на аудиофайл |
| `deliveryStatus` | sending / sent / delivered / failed | Статус доставки |
| `replyTo` | string | ID сообщения, на которое отвечают |
